Model parameters editor

Model parameters editor can be used to add, remove and modify the model parameters of a component. Model parameters are often used in HDL configure component instances. They represent e.g VHDL generics and Verilog module parameters.

All columns of the editor are editable. Double-clicking an empty space adds a new row.

Name column is mandatory. Display name provides a more user-friendly name for the model parameter. Legal data type depends on the language of the model. For example in VHDL this could be 'integer' or 'std_logic', whereas in C-language int or 'char*' could be used. Usage type is mandatory and specifies how this model parameter is used. Default is 'non-typed' since such parameters are found in all languages, i.e. all VHDL types are non-typed. 'Typed' parameters appear in object-oriented languages, i.e. in C++.

Format defines the storage format for the model parameter. Possible formats are:

  • bitString indicates that the the value is a string of 0's and 1's. Optionally, the string can be enclosed in double quotes.
  • bool indicates that the value is either true or false.
  • float indicates that the value is a floating point number. Both standard and scientific format (e.g. 0.002 and 2e-3) are supported. This format applies also to minimum value and maximum value.
  • long indicates that the value is either a long integer or a hexadecimal number. This format applies also to minimum value and maximum value.
  • string indicates that the value shall contain any text.

Bit string length is mandatory, if bitString format is selected. It defines the length of the bit string that stores the value. If any other format is selected, the field must be left empty.

Minimum value and maximum value define the lower and upper boundary for the model parameter value. If the selected format is any other than long or float, these fields have no effect.

By selecting a choice, the user can restrict the allowed model parameter values to a set of predefined values. Possible values are defined in the choices of the containing component.

Value contains the mandatory default value of the model parameter. This value can be overridden in a design that instantiates this component.

Resolve specifies the configuration of the model parameter. Possible resolve values are:

  • immediate indicates that the the value is defined within the containing component. The value cannot be overridden by the user or a generator in design configuration.
  • user indicates that the user can override the default value in design configuration. Generators are not allowed to change the value.
  • generated indicates that the a generator can override the default value in design configuration.

Description is free text for further details.

Array size specifies the array dimension for the model parameter in an output language e.g. Verilog.

Array offset specifies the starting point in the array dimension in an output language e.g. Verilog.

Model parameters editor contains a context menu (rigth mouse button) that has the following options:

  • Add new model parameter (Add row)
  • Remove model parameter (Remove row)
  • Clear the selected cells
  • Copy the contents of the selected cells to clipboard
  • Paste the contents of the clipboard to the selected cells
  • Import a csv-file to the editor
  • Export the contents of the editor to a csv-file
